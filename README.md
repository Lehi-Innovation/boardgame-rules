# Board Game Rules Database

AI-friendly board game rules, built for one job: **asking an AI assistant a
rules question in the middle of a game and getting a trustworthy answer.**

Published site: https://jonnyallred.github.io/boardgame-rules/

## What's in the box

Every game ships as two artifacts plus metadata:

| Artifact | Path | Role |
|---|---|---|
| Rules summary | `rules/<slug>.md` | Structured, LLM-friendly summary with YAML frontmatter |
| Extracted rulebook text | `extracted/<slug>-rules.txt` | The authoritative source the summary was written from |
| Catalog manifest | `games.json` | Machine-readable index: titles, slugs, URLs, verification status |

Summaries are generated by an LLM and then **semantically verified** against
the extracted rulebook text. Every summary page carries a verification banner:

- ✅ **verified / minor_issues** — fact-checked against the rulebook text
- ⚠️ **unverified** — not yet fact-checked (source text available)
- ⚠️ **unverifiable** — no source text available to check against
- ❗ **inaccurate** — an audit found errors that could mislead players; the
  page says so and points at the rulebook text instead

## Using it mid-game

**Simplest:** paste the prompt from the [site landing page](https://jonnyallred.github.io/boardgame-rules/)
into Claude (works great with voice mode at the table). It teaches the
assistant to resolve games via `games.json`, answer from the summary, and fall
back to the full rulebook text for high-stakes questions.

**Better — MCP server:** this repo ships an MCP server that gives any
MCP-capable client direct tools (`list_games`, `get_rules`,
`search_rulebook`, `report_rule_error`):

```bash
pip install mcp
python -m mcp_server.server   # stdio transport
```

Claude Desktop config:

```json
{"mcpServers": {"boardgame-rules": {
  "command": "python",
  "args": ["-m", "mcp_server.server"],
  "cwd": "/path/to/boardgame-rules"}}}
```

## Reporting a wrong rule

Every summary page has a **Report a rules error** link that opens a prefilled
[issue form](https://github.com/Lehi-Innovation/boardgame-rules/issues/new?template=rule-error.yml).
Reports are verified against the extracted rulebook text — never against
memory — and confirmed errors are fixed via pull request. This is how the
database improves every time a table catches a mistake.
