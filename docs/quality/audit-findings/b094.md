# Audit Findings — Batch b094

## fields-of-fire

**Verdict: MAJOR**

---

### Finding 1 — MAJOR: Dice listed as a component; source states the game is diceless

**Summary:** Lists "Dice" as a component in the Components section.

**Source (1.1 Concept, line 175):** "Fields of Fire is a diceless game. Instead, decks of cards are used to generate the battlefield, resolve combats, provide possible outcomes for all of your decisions and control the appearance and activities of the enemy."

The source is explicit. Including dice in the component list is directly contradicted.

---

### Finding 2 — MAJOR: Turn structure is wrong — summary invents a 3-phase sequence not present in the source

**Summary (Turn Structure):** Describes three phases: "1. Initiative Phase," "2. Activity Phase," "3. End Phase."

**Source (Section 3, lines 970–1177):** The actual turn sequence has 7–8 distinct phases:
- 3.1 Friendly Higher HQ Event Phase (starting Turn 2)
- 3.2 Defensive Missions: Enemy Activity Phase (OR 3.4 Offensive Missions & Combat Patrols: Enemy Activity Phase)
- 3.3 Friendly Command Phase
- 3.5 Mutual Capture & Retreat Phase
- 3.6 Mutual Vehicle-Aircraft Phase
- 3.7 Mutual Combat Phase
- 3.8 Clean Up Phase

None of the actual phases is named "Initiative Phase," "Activity Phase," or "End Phase." The summary's sequence is entirely invented.

---

### Finding 3 — MAJOR: Activity Phase resolution order is invented

**Summary (Activity Phase):** "Resolve activities in order: friendly fires, enemy fires, movement, close combat."

**Source:** No such unified "Activity Phase" exists. Movement is ordered during the Friendly Command Phase (3.3) via Command Points and Actions. Enemy activity occurs in a separate Enemy Activity Phase (3.2 / 3.4). Infantry combat takes place in the Mutual Combat Phase (3.7). There is no phase that combines all four of these elements together in the stated order. The order "friendly fires, enemy fires, movement, close combat" does not appear in the source under any turn-sequence description.

No support found (searched: "friendly fires," "enemy fires, movement," "resolve activities in order," "close combat" in turn sequence context).

---

### Finding 4 — MAJOR: "End Phase" checking morale and objectives is invented

**Summary (End Phase):** "Check morale and casualties. Update command and control status. Check mission objectives."

**Source (3.8 Clean Up Phase, lines 1171–1177):** "Remove Pyrotechnic, Illumination, Exposed, Moved/Fired, Concentrated Fire, Grenade, and Grenade Miss Markers. Casualties dropped off at a Terrain Card with a designated Casualty Collection Point Tactical Control Marker on it are evacuated in WWII and Korean War Missions. During Defensive Missions, remove any unresolved PC Markers."

The actual Clean Up Phase only removes markers. There is no "morale check," no C2 status update step, and no objective check in the turn's End Phase. The term "morale" does not appear as a per-turn phase check in the source (searched: "morale," "check morale," "end phase").

---

### Finding 5 — MAJOR: Named orders list (Advance, Attack, Defend, Withdraw, Fire Mission) is invented

**Summary (Actions/Orders):** "Advance: Move toward objectives / Attack: Engage enemy positions / Defend: Hold current position / Withdraw: Fall back / Fire Mission: Call in artillery/mortar support"

**Source (Sections 4.1–4.2, lines 1348–1740):** The game does not use named order types called "Advance," "Attack," "Defend," "Withdraw," or "Fire Mission." Instead, HQs expend Command Points to issue specific Actions from the Action Menu (e.g., "Move to an Adjacent Card," "Attempt to Infiltrate," "Attempt to Concentrate Fire," "Attempt to Spot," "Call for Fire from an FO," etc.). The source says: "The core mechanic of Fields of Fire is that HQs obtain a number of Commands and use them for Actions to control themselves and other units." Commands are traded for specific listed Actions, not the broad order types the summary describes.

No support found (searched: "ADVANCE," "ATTACK," "DEFEND," "WITHDRAW" as order types in section 4; "Fire Mission" as an order type). Note: "ADVANCE" and "CEASE FIRE" do appear as commands linked to pyrotechnic signals (4.3.6), not as order types assigned to units during normal play.

---

### Finding 6 — MAJOR: Victory conditions described as "Major Victory to Major Defeat" ratings — no such rating scale in source

**Summary (Scoring / Victory Conditions):** "Missions are rated from Major Victory to Major Defeat."

**Source (Section 11, lines 4698–4810):** "You must successfully complete a mission before moving on to the next mission in the campaign. The Mission Instructions will indicate what constitutes the successful completion of a mission and the number of times the mission may be attempted." The source describes only binary success/failure. Section 11.2 states: "If you cannot complete a Mission successfully within the number of attempts given by the Mission Instructions, you fail the Campaign." No rating scale from Major Victory to Major Defeat appears anywhere in the rules text (searched: "major victory," "minor victory," "major defeat," "minor defeat," "marginal" — no matches found).

---

### Finding 7 — MINOR: Pyrotechnic description is inaccurate but not table-breaking

**Summary (Special Rules):** "Pyrotechnics: Colored smoke can mark positions for air/artillery support."

**Source (4.3.6, lines 1981–1993):** "Pyrotechnic devices have three basic purposes: Signaling, Illuminating, and Concealment." The specific commands linked to pyrotechnics are CEASE FIRE (Red), ADVANCE toward Primary Objective (Green), and SHIFT FIRE (Yellow/Purple). The purpose is commanding units via pre-assigned signals, not marking positions for air/artillery support. No source support found for the "mark positions for air/artillery support" characterization (searched: "mark position," "air support smoke," "artillery smoke marking"). This is a misleading simplification but less immediately table-breaking than the structural errors above.

### Summary of issues

The summary has multiple MAJOR errors that would prevent play:
1. Dice listed as a component — source is explicitly diceless
2. Turn structure entirely wrong (3 invented phases vs. 7–8 actual phases)
3. Resolution order within the invented "Activity Phase" is unsupported
4. "End Phase" morale/objective checking is invented
5. Order types (Advance/Attack/Defend/Withdraw/Fire Mission) are invented — the game uses Command Points spent on specific Actions from an Action Menu
6. Victory rating scale (Major Victory to Major Defeat) is invented — source only has binary success/failure

A player using this summary would not be able to run a turn correctly.
