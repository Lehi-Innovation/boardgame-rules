## power-grid

### Verdict: MINOR

### Prefilter
OK — source is the Power Grid rulebook by Friedemann Friese.

### Step 1 — Summary claims checked
- 5 phases, player count table, step triggers, auction rules, resources, victory condition, tiebreakers.

### Step 2 — Source survey
Source fully read (309 lines). Key sections: Winning the Game (lines 252-255), Steps of the Game (lines 220-247), Phase 2 Auction (lines 66-96), player-count exception table (lines 259-300, table values OCR-garbled).

### Step 3 — Verification

#### Victory & game-end conditions
Summary: "The game ends during Phase 5 when a player connects to a target number of cities (varies by player count)."
Source (line 253): "The game ends immediately after phase 4 when at least one player has at least 17 cities in his network."

The summary says the game ends during Phase 5 (Bureaucracy). The source says it ends "immediately after phase 4" (Building). This is a phase-timing error, though its practical effect is limited (it triggers before Phase 5 begins).

Summary player-count table: "2=21, 3=17, 4=17, 5=15, 6=14."
Source: The player-count table exists (line 296-299: "Number of connected cities to trigger game end") but its numeric values are not readable in the extracted text (OCR produced only header text, no data cells). The only explicitly stated threshold is 17 (line 253). The summary's table cannot be confirmed or denied from the source.

#### Tiebreaker
Summary: "Tiebreaker: most Elektro remaining."
Source (lines 254-255): "If there is a tie, the player with the most remaining money wins. If there is still a tie, the player with the most cities in his network is the winner."
The summary omits the secondary tiebreaker (most cities). This is a MINOR omission.

#### Max power plants per player
Summary: "Max 3 power plants per player."
Source (line 87): "During the game each player can have only 3 power plants at any time."
Source (lines 260-261): "For example you need fewer power plants for 2 players, but both of them can own up to 4 power plants."
The summary's blanket statement of 3 is incorrect for 2-player games where players can own up to 4. This exception is noted in the Special Rules section only as "Ecological plants require no resources" — the 2-player exception is omitted.

#### Step 2 trigger
Summary: "Step 2 begins when any player connects to a threshold number of cities."
Source (line 226): "Step 2 starts before phase 5 (bureaucracy) after a player has built his 7th city during phase 4 (Building)."
The summary does not specify the 7-city threshold.

#### Phase order
Summary Phase 5 Bureaucracy: "In player order, power cities using fueled plants. Earn income based on cities powered. Refill resource market based on step and player count. Discard lowest plant if no one bought this round."
Source (lines 195-218): Bureaucracy correctly includes powering cities (earning cash), re-supplying resource market, and updating the power plant market. The summary incorrectly states "Discard lowest plant if no one bought this round" — the source says the highest-numbered plant from the future market is placed under the draw stack each round (Phase 5), and the lowest plant is only removed if no plant was sold this round (Phase 2, line 95-96). This conflates two different rules.

#### Numbers spot-checked
- Starting Elektro: 50 (source line 22: "50 Elektro") ✓
- Power plant cards: 43 (summary) vs source line 8: "43 power plant cards: 42 power plants with numbers 03–40, 42, 44, 46 and 50" ✓
- Resource market setup (coal, oil, garbage, uranium): match source lines 24-28 ✓
- Step 3: cities accept 3 houses, 20 Elektro for 3rd house: source line 244 ✓
- Building cost: 10 for 1st, 15 for 2nd, 20 for 3rd house (source line 163) ✓

NUMBERS_CHECKED: 8, SUPPORTED: 6 (2 unverifiable/problematic)

### Summary of findings
1. (MINOR) Game-end trigger timing: summary says Phase 5, source says immediately after Phase 4.
2. (MINOR) Tiebreaker incomplete: missing secondary tiebreaker (most cities).
3. (MINOR) Max plants per player: blanket "3" misses 2-player exception (up to 4).
4. (MINOR) Step 2 trigger: threshold (7 cities) omitted.
5. (MINOR) Phase 5 Bureaucracy: "Discard lowest plant if no one bought" conflates Phase 2 rule with Phase 5 plant market update.

No single finding rises to MAJOR (no scoring numbers are wrong, no invented mechanics, the core win condition — most powered cities — is correct). The errors are omissions and mild timing/phrasing inaccuracies.

## power-grid-factory-manager

### Verdict: MAJOR

### Prefilter
OK — source is the Factory Manager rulebook by Friedemann Friese.

### Step 1 — Summary claims checked
- 5 rounds, income formula, tiebreaker, 4-phase structure in Player Reference.

### Step 2 — Source survey
Source fully read (282 lines). Key sections: Playing the game (lines 131-265), Income phase (lines 249-261), Game End (line 265).

### Step 3 — Verification

#### Phase count error (Player Reference omits a phase)
Summary Player Reference lists 4 phases:
1. Player Order
2. Market (Spend workers to buy factory tiles)
3. Operations (Run factory; earn revenue minus energy)
4. Energy (Adjust energy price)

Source (lines 132-138) lists 5 phases:
1. Auction the new player order
2. Preparing the market
3. Buying & placing factory tiles, tearing down factory tiles, and hiring seasonal workers
4. Bureaucracy and new energy price
5. Distribute income

The summary collapses Phase 5 (Income) into Phase 3 (Operations), and Phase 4 (Bureaucracy which includes adjusting factory tracks, canteen workers, AND energy price) is described only as "Adjust energy price." This causes the Player Reference to be incorrect in phase count and ordering.

#### Income formula (MAJOR)
Summary Phase 3 says: "Revenue = production value - energy costs."
Source (lines 249-256): "The income is based on the earnings (the lower number of the production or storage) minus the actual energy costs... The earnings for a player is the lower of his production and storage tracks."

The summary uses "production value" as income base; the source uses MIN(production, storage). A player with production=10 and storage=2 earns on 2, not 10. This wrong formula would cause players to grossly miscalculate their income every round and is a table-breaking error.

Summary example: source gives example "production of 3 and storage of 8 will have an income of only 30 Elektro - the value of his production, which is lower than his storage" — confirming the min(production, storage) rule. The summary's formula is definitively wrong.

#### Final round income doubling
Summary: no mention.
Source (line 261): "During the fifth and final round, the income of all players is doubled!"
This major scoring rule is completely absent from the summary. It directly affects the final score for every player.

#### Tiebreaker
Summary: not stated.
Source (line 265): "In case of a tie the player with a higher income in the last turn wins the game."
The tiebreaker is absent from the summary.

#### Phase 2 "Preparing the market" mechanics
Summary: describes "In player order, each player uses their workers to acquire tiles from the central market. Workers are your currency for buying tiles."
Source (lines 162-173): Phase 2 is "Preparing the market" — players select tiles FROM the storage board TO the market area (using available workers). Phase 3 is actually buying tiles from that market. The summary conflates Phase 2 and Phase 3 and places the combined action in "Phase 2: Market Phase."

#### Numbers spot-checked
- 5 rounds: ✓ (source line 132)
- 7 workers per player: ✓ (source line 35)
- 2 seasonal workers per player: ✓ (source line 37)
- 5 factory boards: ✓ (source line 20)
- 106 factory tiles: ✓ (source line 25)
- Starting money: summary does not state it; source line 103 = 18 Elektro (omission in summary)
- Energy price starts at 1 Elektro: source line 120 confirms

NUMBERS_CHECKED: 7, SUPPORTED: 5 (income formula and final-round doubling wrong/absent)

### Summary of findings
1. (MAJOR) Income formula wrong: summary uses "production value" as base; source requires MIN(production, storage).
2. (MAJOR) Final-round income doubling absent: every player's Round 5 income is doubled (source line 261); entirely missing from summary.
3. (MAJOR) Phase count wrong: summary has 4 phases; source has 5, with Phase 2 "Preparing market" omitted from Player Reference.
4. (MINOR) Tiebreaker absent.
5. (MINOR) Starting money (18 Elektro) not stated.

## powerline

### Verdict: MAJOR

### Prefilter
OK — source is the Powerline rulebook by Dirk Henn (Queen Games).

### Step 1 — Summary claims checked
- 16 rounds stated, 5-phase turn structure described, scoring tile values, game-end penalties.

### Step 2 — Source survey
Source fully read (532 lines).

### Step 3 — Verification

#### Round count (MAJOR)
Summary: "The game is played over 16 rounds."
Source (line 147): "Powerline is played over 15 game rounds."
Source (lines 317-318): "When the round marker lands on one of the 3 extra scoring spaces – after the 5th, 10th, and 15th round."

The summary states 16 rounds; the source is explicit about 15. This directly affects game length and scoring timing.

#### Turn structure / phase count (MAJOR)
Summary describes 5 phases per round:
1. Roll Dice
2. Choose Direction (left-to-right or right-to-left)
3. Use Dice (build powerlines)
4. Workers and Penalties
5. Scoring Checks

Source (lines 153-156) gives 3 phases per round:
1. Roll Dice
2. Plan Workers
3. Construct Powerlines

The summary invents "Choose Direction" as a separate phase 2. In the source, direction choice is part of Phase 2 (Plan Workers — players must start from red or black and proceed in color order, which can go left-to-right or right-to-left). "Workers and Penalties" and "Scoring Checks" are not discrete phases; they are elements within Phase 2 and post-round resolution respectively. The summary's phase structure is fabricated relative to the source.

#### Scoring structure
Summary: "During certain rounds, the 3 game objectives are evaluated and scored."
Source (lines 317-318): Scoring occurs after rounds 5, 10, and 15 (not "certain rounds" vaguely). This is imprecise but not a table-breaking error.

Summary scoring tiles (A, D, G) match source values:
- A) Cities: 1st scoring: 2VP for 1-2 cities, 5VP for 3-5 cities ✓ (source line 333)
- A) Cities: 2nd scoring: 1VP for 1-2, 3VP for 3-5, 5VP for 6+ ✓ (source line 334)
- A) Cities: 3rd scoring: 0VP for 1-2, 1VP for 3-5, 3VP for 6+ ✓ (source line 335)
- D) Power station: 1st scoring: 4VP/1 tile, 5VP/2 tiles, 7VP/3 tiles ✓ (source line 337)
- G) Jokers: 1VP each / 2VP each / 3VP each ✓ (source line 342)

#### End-of-game penalty
Summary: "Penalty points for started but uncompleted lines."
Source (lines 325-329): Loss of VP per number of unfinished powerlines (table shows [+1] and [+5] thresholds). The summary does not state the penalty amount, which is an omission but not a wrong value.

#### Island penalty
Summary: "Skipping a die incurs a penalty."
Source (lines 206-209): "For every worker on the island, lose 1 VP." The summary doesn't specify the 1 VP cost but the general concept is correct.

#### VP starting position
Summary: no mention.
Source (lines 122-125): "Place it on space 10 of the VP-track on the center board." All players start at 10 VP.

NUMBERS_CHECKED: 8, SUPPORTED: 6 (round count wrong, phase count wrong)

### Summary of findings
1. (MAJOR) Round count: summary says 16 rounds; source says 15 rounds.
2. (MAJOR) Turn structure: summary describes 5 phases per round (inventing "Choose Direction" and "Scoring Checks" as phases); source has 3 phases (Roll Dice, Plan Workers, Construct Powerlines).

## primordial-soup

### Verdict: MAJOR

### Prefilter
OK — source is the Primordial Soup (Ursuppe) rulebook.

### Step 1 — Summary claims checked
- 6-phase structure, eating rules, excretion, reproduction cost, game-end condition, tiebreaker.

### Step 2 — Source survey
Source fully read (408 lines).

### Step 3 — Verification

#### Phase order — MAJOR (phases 3, 4, 5 scrambled)
Summary phases:
1. Movement and Eating ✓
2. Environment and Gene Defects ✓
3. Death
4. Division (Reproduction)
5. Buying Genes
6. Scoring ✓

Source phases (lines 142-216):
1. Movement and Feeding ✓
2. Environment and Gene Defects ✓
3. New Genes (Buy Genes) ← summary places this as Phase 5
4. Cell Division ← summary places this as Phase 4 (correct position but in wrong context)
5. Deaths ← summary places this as Phase 3
6. Scoring ✓

The summary has reversed Phases 3 and 5: amoebas die (Phase 3 in summary) before players can buy genes to prevent deaths (Phase 5 in summary). In reality, players buy genes (Phase 3) and reproduce (Phase 4) BEFORE amoebas die (Phase 5). This is a table-breaking reversal — players using the summary would lose amoebas to death before having the opportunity to spend BPs on genes or reproduction that round.

Quote from source (line 109): "PHASE 3: New Genes — In Descending Order, each player can use BPs to buy new Genes."
Quote from source (line 207): "PHASE 5: Deaths — Amoebas that have 2 or more Damage Points die..."

#### Eating rules — MAJOR (wrong number of cubes consumed and excreted)
Summary: "the amoeba must eat: consume 2 foodstuff cubes of different colors (not your own color)"
Summary Player Reference: "Move (1 BP/space), eat 2 cubes, excrete 1 cube"
Summary: "After eating, excrete 1 foodstuff cube of your own color onto your space."

Source (lines 180-186): "An amoeba eats three Foodstuff cubes per turn, but never one of its own color... If an amoeba finds enough food, the food that it has eaten is replaced by two cubes of the amoeba's own color."
Source (lines 291-294): "Feeding: An amoeba without any special capability eats 3 Foodstuff cubes per turn... Once it has eaten, it excretes two cubes of its own color."

The summary states 2 cubes eaten and 1 excreted. The source states 3 cubes eaten and 2 excreted. Both numbers are wrong. This affects food availability calculations and starvation outcomes in every turn.

#### 3-player eating
Summary: "eat: consume 2 foodstuff cubes of different colors"
Source (lines 182-184): "In a three player game, it eats one Foodstuff cube of one color and two of another color."
Summary does not distinguish 3-player vs 4-player eating. In 4 players, each amoeba eats 1 of each of the other 3 colors. In 3 players, it eats 2 of one color and 1 of another.

#### Division cost
Summary: "pay BP (cost depends on Gene Cards)."
Source (lines 120-122): "A player first gets 10 BPs. Using these, plus any that they may have saved from earlier rounds, they may cause their amoebas to divide. At a price of 6 BPs per amoeba..."
The base cost is 6 BP per amoeba. The summary omits this specific number and the 10 BP grant. It also does not mention that players receive 10 BPs at the start of Phase 4.

#### Game-end condition
Summary: "The game ends after a set number of rounds (or when Environment Cards run out)."
Source (lines 240-247): "If, after all markers have advanced, there are no markers in the dark Finish Zone of the Scoring Ladder, the game enters another round... if one or more markers have reached the dark Finish Zone at the end of the Scoring Ladder, the game ends."
The primary end condition is reaching the Finish Zone on the Scoring Ladder — not "a set number of rounds." The summary inverts the primary and secondary end conditions.

#### Tiebreaker
Summary: "The player with the most points on the scoring ladder wins."
Source (line 248-249): "The player whose marker is furthest along the Scoring Ladder wins the game." ✓

#### Scoring advance table
Summary: "Each player scores points based on living amoebas and Gene Cards owned."
Source (lines 218-233): Exact advance table — e.g., 0-2 amoebas = 0 spaces, 3 amoebas = 1 space, 4 = 2, 5 = 4, 6 = 5, 7 = 6; gene cards 0-2 = 0, 3 = 1, 4 = 2, 5 = 3, 6+ = 4. Also: Advanced Genes count as 2 cards; RAY PROTECTION does not count. Summary completely omits the specific advance table — without it, players cannot score.

NUMBERS_CHECKED: 5, SUPPORTED: 1 (eating counts, excretion count, division cost 6 BP, 10 BP grant, advance table all wrong or absent)

### Summary of findings
1. (MAJOR) Phase order scrambled: Death (Phase 5) moved to Phase 3; Buy Genes (Phase 3) moved to Phase 5. Players would lose amoebas before buying genes or reproducing.
2. (MAJOR) Eating: summary says 2 cubes eaten; source says 3. Excretion: summary says 1 cube; source says 2.
3. (MAJOR) Game-end: summary says "set number of rounds"; primary trigger is actually reaching the Finish Zone on the Scoring Ladder.
4. (MAJOR) Scoring advance table entirely absent — cannot score a game using the summary.
5. (MINOR) Division cost (6 BP) and 10 BP grant in Phase 4 absent.
6. (MINOR) 3-player vs 4-player eating distinction absent.
