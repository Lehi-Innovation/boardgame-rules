## on-mars

### Verdict: MINOR

The summary correctly captures the core game structure (Colonization Phase + Shuttle Phase, Orbit vs Colony actions, OP win condition, 3-mission game-end trigger). The game-end conditions are supported by the source.

**Victory / game-end verification:**

- Summary: "Game ends after a set number of rounds (determined by player count and variant)"
- Source (p. overview): "Once a total of three missions have been completed, the game ends." — The summary's description of game end as "a set number of rounds" is inaccurate; the primary end trigger is 3 missions completed (or Colony reaching Level 5, or Remaining Missions marker logic). The summary does mention missions in the scoring section but doesn't clearly state that completing 3 missions is the game-end trigger.
  
  However the summary does say: "Game ends after a set number of rounds (determined by player count and variant)" which contradicts the source. That said, this is in the context of a high-level overview and the game does involve variants. But the actual end trigger is missions, not a fixed number of rounds. This is a misleading omission of the primary game-end mechanism.

- Source: "Once a total of 3 Missions have been completed collectively by all players, the end of the game is triggered" and "At Colony level 5, the end of the game is triggered, even if no Missions have been completed."

**Scoring sources verification:**
- Completed Missions: supported (source: "Crystals shown on the card" during game, plus game-end OP)
- Private Goals: supported (source: "completing a Private Goal" grants rewards)
- Buildings: supported (source: "OP for Advanced Buildings, 3 points for Lvl 1 card and 5 points for Lvl 3 cards")
- Scientists: supported (source: "Each Scientist scores 3 OP for each Advanced Building of the indicated types on Mars")
- Earth Contracts: supported (source: "Gain or lose OP for each Earth Contract card as depicted on the card")
- Colony advancement: supported (source: Progress cubes, Colony Status updates)
- Resource and technology bonuses: supported (source: Tech tiles worth OP below their column)

**Tiebreaker:**
- Source: "In the case of a tie, the player with the most Crystals is the winner. If there is still a tie, the player with the most advanced buildings is the winner. If there is still a tie, the player with the most Progress cubes in the Progress area is the winner."
- Summary: Does not mention any tiebreaker. MINOR omission.

**Procedural checks (5 claims):**
1. "Orbital Station Actions: Landing Pod / Blueprint / Technology / R&D / Resupply" — supported by source Table of Contents listing these as Orbital Station Main Actions.
2. "Colony Surface Actions: Control Center / Construct / Upgrade / Welcome Ship / Hire Scientist or Earth Contract" — supported by source Table of Contents.
3. "Executive Actions: Travel / Boost Tech / Scientists / Complete Missions" — supported by source.
4. "Shuttle Phase: Ships travel between Earth and Mars" — supported by source.
5. "LSS must maintain balance of water, oxygen, power, food" — supported (source: "carefully balancing the importance of water, air, power, and food").

**Numbers checked:**
- The summary lists no specific numbers in scoring, making numbers verification limited. OP values from the source (1/2/4/7/11 for progress cubes, 3 OP per ship, 3/5 OP for blueprints) are not in summary — omission, not contradiction.

**Main finding:** The summary says "Game ends after a set number of rounds (determined by player count and variant)" which is inaccurate — the primary end trigger is mission completion (or colony level), not rounds. This could mislead players about when the game ends, but given the summary does discuss missions, it's a MINOR rather than MAJOR issue.

NUMBERS_CHECKED: 0 specific numbers in scoring section, SUPPORTED: n/a (summary gives no specific OP values)

## onward

### Verdict: MAJOR

**Finding 1: Missing day-5 fallback win condition (MAJOR)**
- Summary: "Win: Destroy the enemy Nexus by reducing its 9 hit points to 0." / "There is no point-based scoring -- the game is won by destroying the opposing Nexus."
- Source: "At the end of the fifth Day, the team whose nexus has most HP wins (or the team with priority, in case of tie)." 
- The summary omits the fallback win condition entirely. If no nexus is destroyed by end of Day 5, the team with more nexus HP wins. Players using only the summary would not know the game can end this way.

**Finding 2: Minions do not advance automatically (MAJOR)**
- Summary: "Minion waves advance automatically each round."
- Source (Minion Phase): Minions advance only when they have excess momentum and cannot assault or siege. The phase involves counting momentum per team, spending the difference on actions (assault, siege, advance), and placing one minion each. Source: "Spend 1 momentum to move all of your minions...into an adjacent lane area toward the enemy nexus." Minions only advance when momentum allows — this is a complex momentum-based system, not automatic.
- This would fundamentally mislead players about how minion pressure works.

**Finding 3: Monsters grant gold only, not "gold and experience"**
- Summary: "Slay Monsters: Defeat jungle monsters for gold and experience"
- Source: "defeat monsters to earn gold, which they use to buy powerful items and grow stronger." — No "experience" mechanic appears in the source. Leveling up heroes comes from defeating enemy heroes (not monsters). Grepping "experience" in the source finds no game mechanic named experience.
- No support found (searched: "experience", "XP", "level.*monster", "monster.*level").

**Finding 4: Units called "champions" throughout summary, source uses "heroes" exclusively**
- Summary: "Champion cards and miniatures," "Each player selects champions through a drafting process," "Champions that are defeated respawn..."
- Source: "hero cards," "hero figures," "Each team chooses 4 different heroes." The source never calls them "champions" in the game rules (only in lore flavor text on last page). This is wrong terminology throughout but less critical than findings 1-3. Noted as corroborating evidence of the summary's inaccuracy.

**Procedural checks (5 claims):**
1. "Nexus boards (9 HP each)" — source: "Each team places their nexus token next to the number 9 of the hit points (HP) track." — MATCHES.
2. Game is "competitive" between Light and Shadow teams — source confirms teams: Light and Shadow — MATCHES.
3. "Draft champions" — source does describe optional drafting methods, so this is supported though simplified. SUPPORTED.
4. "Respawn after a set number of turns" — source: heroes become exhausted, then respawning, then re-enter play via planning — approximately supported but the mechanism is more complex than "set number of turns."
5. Round structure: Shop Phase → Plan Phase → Hero turns → Minion Phase → Jungling Phase → Onward Phase — source confirms this structure — MATCHES.

**Numbers:**
- Nexus 9 HP — MATCHES.
- 2-4 players — source: "2-4" — MATCHES.

NUMBERS_CHECKED: 2, SUPPORTED: 2

## ora-et-labora

### Verdict: MAJOR

**Finding 1: "Buy a Landscape" listed as a main action — it is an Extra Action (MAJOR)**
- Summary Turn Structure: "On each turn, a player must perform exactly one of the following actions: 1. Cut Peat / Fell Trees, 2. Build a Building, 3. Use a Building, 4. Buy a Landscape"
- Source (The Action Options): "Phase 4 of a round is the action phase. You have **three** options for this action. You can place one of your own clergymen (prior or lay brother)...You can fell trees or cut peat...You can build a building."
- Then separately: "**Extra Action** Buy a landscape — You may buy a landscape once per turn and once per settlement phase."
- The summary promotes "Buy a Landscape" from an Extra Action (available in addition to a main action) to one of the three mutually exclusive main action choices. Players following the summary would never buy a landscape on the same turn as another action, fundamentally misplaying a core mechanic.

**Finding 2: The Player Reference table also repeats the error**
- Summary Player Reference: "Turn: Choose 1 action: Cut Peat / Fell Trees / Build / Use Building / Buy Landscape"
- Source: Three main action options + "Buy landscape" as Extra Action. Same error reinforced.

**Procedural checks (5 claims):**
1. "Production wheel advances each round; resources accumulate" — source: "rotate the production wheel one position in the direction of the arrow illustrated on it (i.e. counterclockwise). The goods indicators indicate production levels." — MATCHES.
2. "Prior (flexible), Lay Brother (restricted)" — the source's extracted text does not explicitly describe Lay Brother restrictions beyond the prior's special ability to immediately use a newly-built building. The source states: "The effect of the prior when building: After having built a building, you can use your prior to immediately use the function of the building." No restrictions on Lay Brother for building use are stated in the extract. The summary's claim that "the Lay Brother has restrictions on which buildings he can use" has no support found in the extracted text (searched: "lay brother restrict", "lay brother cannot", "only prior").
3. "France (wine/bread) or Ireland (whiskey/peat)" — source confirms: "France variant" has wine/bread/grapes and "Ireland variant" has whiskey/peat/malt — MATCHES.
4. "You may use your own buildings for free or pay another player to use theirs" — source: "If you want to use one of an opponent's buildings, then you must pay him 1 coin and, by doing so, issue a work contract" — MATCHES (payment required).
5. "Settlements score based on surrounding buildings / Dwelling value" — source: "Settlement points are calculated by adding the dwelling values of the settlement...and all adjacent buildings together." — MATCHES.

**Scoring verification:**
- Summary: "Each building has a printed VP value" — source: "score the economic value of all buildings and settlements" — MATCHES in intent.
- Summary: "Coins count as VP" — source: "5 coins: 2 [points], 1 coin: subsumed in goods scoring" — MATCHES.
- Summary: "Settlements score based on surrounding buildings" — source: "Settlement points are calculated by adding the dwelling values of the settlement and all adjacent buildings together." — MATCHES.
