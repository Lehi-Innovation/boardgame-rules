# Semantic Audit of Rules Summaries

You are auditing AI-generated rules summaries (`rules/<slug>.md`) against their
source rulebook text (`extracted/<slug>-rules.txt`) in this repository. The
summaries were generated by an LLM and **roughly half contain serious errors**.
Your job: find them, with evidence, and emit a verdict per game.

You will be given a **batch id** (e.g. `b073`) and a list of **slugs**.

## Non-negotiable rules

1. **Never use outside knowledge of the game.** Only the extracted text counts
   as truth. If you know the "real" rules from training data but the source
   says otherwise, the source wins. (Summaries fail audits precisely because
   their author used outside knowledge.)
2. **Every finding needs evidence**: a short quote from the summary AND either
   a short quote from the source that contradicts it, or the statement
   "no support found (searched: <the grep terms you tried>)".
3. **Only report findings you are confident in.** The extracted text has OCR
   artifacts, merged columns, and lost tables — search with several alternate
   terms before declaring something unsupported. An uncertain finding is worse
   than no finding: drop it.
4. **Correct numbers do NOT mean a correct summary.** The most damaging errors
   found so far were in *procedures and relationships*, not numbers: summaries
   with 20+ verified numbers have still been unplayable. Never upgrade a
   verdict because the numbers check out.

## Procedure (per game, in this order)

### Step 0 — Prefilter
Run once for the whole batch:
```bash
python -m scripts.audit prefilter <slug1> <slug2> ...
```
- `MISSING_EXTRACTED` → emit `RESULT|<slug>|MAJOR|No extracted source text to audit against` and skip the game.
- `SOURCE_MISMATCH` / `NOT_RULES` → confirm by reading the first ~50 lines of
  the extracted file. If the source really is a different game or not a
  rulebook (product page, article, web scrape), emit
  `RESULT|<slug>|MAJOR|SOURCE_MISMATCH: <what the source actually is>` and skip
  the detailed checks — the summary cannot be verified and was necessarily
  written from outside knowledge.

### Step 1 — Read the summary fully
It is short. Note every checkable claim as you read: victory/end conditions,
turn structure, action lists, costs, component counts, scoring tables, the
Player Reference section.

### Step 2 — Survey the source
If the extracted file is under ~40 KB, read it fully. If larger, read the
first ~200 lines (title, components, setup), then use Grep with `-C 5` to pull
the sections you need (`victor`, `wins`, `game end`, `score`, `phase`,
`turn`, plus game-specific terms from the summary).

### Step 3 — Verify in priority order
Work down this list; it is ordered by how often summaries get each one wrong:

1. **Victory & game-end conditions** — the single most common serious error.
   Check: the win condition itself, every game-end trigger, exact thresholds,
   tiebreakers (often invented or reversed), and per-faction victory in
   asymmetric games (COIN/GMT wargames are wrong more often than right).
   **A matching headline sentence is not verification.** For every term the
   victory condition depends on ("control", "majority", "connected",
   "occupied"…), find the source's *definition* of that term and check the
   summary's definition matches — contradictions usually hide in the
   definition subsection (e.g., a source where "control" means markers placed
   only on *opponent* areas, while the summary lets you count your own).
2. **Invented mechanics** — phases, actions, components, or subsystems in the
   summary that do not exist in the source. Grep the source for the mechanic's
   key noun; if absent under several names, it is invented.
3. **Family contamination** — if the game is a variant, spinoff, junior
   edition, or scenario of a famous game (Catan, Agricola, Monopoly, base
   Civilization…), explicitly check that base-game costs/rules did not replace
   the variant's actual ones. This class fails almost every time.
4. **Direction/order reversals** — who moves first, which player an effect
   targets, attacker vs defender, clue-giver vs guesser, "immune when
   performing X" vs "immune to X". Quote both sides exactly.
5. **Core-system omissions** — entire phases, economies (income, hand limits,
   upkeep), or scoring sources missing. Test: could you score a finished game
   using only the summary? If a scoring source in the source's end-of-game
   section is absent from the summary, that is MAJOR.
6. **Numbers** — spot-check at least 10 (all of them if fewer): costs,
   counts, VP values, thresholds, movement rates. Prioritize the Components
   table and Scoring section.
7. **Player Reference tables** — mappings get scrambled (wonder→effect,
   animal→elements, power→name). Verify each row, not just a sample.

### Step 3.5 — Minimum verification quota (mandatory)

Shallow skims produce false PASSes. For **every** game you must, at minimum,
explicitly verify each of the following by locating the matching source
passage and writing the quote pair into your findings file:

- **Every sentence** of the summary's victory / game-end / tiebreaker rules.
  Find the source's own game-end section (grep `win`, `wins`, `victor`,
  `game end`, `most points`) and compare it against the summary in BOTH
  directions: summary claims missing from the source AND source conditions
  missing from the summary.
- **Every number** in the Scoring / Victory Conditions section.
- **At least 5 procedural claims** from Turn Structure/Actions (sequence,
  who acts, what an action costs/does) — not component counts, which are
  almost always right and prove nothing.
- **Every "term of art" the victory/scoring rules depend on** (control,
  ownership, adjacency…): locate the source's definition and quote it next to
  the summary's usage.
- **Exhaustion rules**: grep the source for what happens when a deck, bag, or
  supply runs out (`empty`, `run out`, `exhaust`, `reshuffle`, `last card`).
  Summaries frequently *invent* a reshuffle/replenish rule the source never
  states; if the summary states one, it must have a quote or it is MAJOR.

If you cannot complete the quota for a game, say so in the findings file
rather than guessing a verdict. Base rate for context: roughly **half** of
the summaries in this catalog contain MAJOR errors, and they cluster in
exactly these sections. If you are about to mark a complex war/euro game
PASS, re-check its game-end and turn-sequence claims one more time first.

### Step 4 — Verdict
- **MAJOR** — any of: a contradicted rule, an invented mechanic, a wrong
  number in scoring/costs, a missing game-end condition or scoring source, a
  scrambled reference table, or a source mismatch. Anything that would mislead
  players at the table.
- **MINOR** — only omissions or unsupported flavor details that would not
  change how the game is played or scored (a missing variant, an uncheckable
  frontmatter field, a small edge case).
- **PASS** — nothing found beyond trivial wording drift. Do not force
  findings; simple games frequently pass.

When torn between MINOR and MAJOR, ask: "would a table of players get a rule
or the final score wrong because of this?" Yes → MAJOR.

## Output contract

1. Write detailed findings to `docs/quality/audit-findings/<batch-id>.md`:
   one section per game with the verdict, each finding as
   summary-quote vs source-quote (or "no support found"), and a
   `NUMBERS_CHECKED: n, SUPPORTED: n` line. Do NOT modify any other file.
2. Your **final message must be ONLY** these lines, one per game:
   ```
   RESULT|<slug>|PASS or MINOR or MAJOR|<one-line reason, or "-" for PASS>
   ```
   The reason must be specific (name the rule and both values), under ~250
   chars, and contain no `|` characters.

## Worked example (real finding)

> Summary (barbarossa): "5-6 guesses: 0 points, 7-8 guesses: +1 point"
> Source: "if 5 or 6 → +1 … if 7 or 8 → +2 … 12 or 13 → −2"
> Finding: scoring table misaligned by one bracket and the 12-13 penalty was
> dropped. Verdict: MAJOR — every score computed from the summary is wrong.

## Failure patterns already confirmed in this catalog (look for these)

| Pattern | Example found |
|---|---|
| Wrong/invented victory condition | "270 electoral votes" vs source "269 of 537" |
| Invented action | a "Pass" action in a game whose rules forbid passing |
| Base-game contamination | Catan variant given base-Catan 4:1 trade and desert hex |
| Reversal | "defender rolls first" vs source "homeland side rolls first" |
| Scrambled table | wonder effects assigned to the wrong wonders |
| Source mismatch | summary of classic Connect Four; PDF is "Connect 4 & More" |
| Fabricated metadata | designers credited from example-player names in the rulebook |
| Invented exhaustion rule | "shuffle your discard pile into a new deck" with no source support, in a game paced by deck depletion |
| Headline matches, definition doesn't | "most Victory Areas wins" matched, but source defines control via markers placed only on opponent areas |
